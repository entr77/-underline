## 0. 핵심 포지셔닝 — 왜 우리가 유리한가

**대부분의 신청자**: "AI를 도입하려고 합니다"
**우리**: "Claude AI를 이미 프로덕션에서 운영 중이고, 고도화 자금이 필요합니다"

이 차이가 모든 평가 항목에서 유리하게 작동한다.
- AI활용 적합성 → "계획"이 아닌 "실적"으로 답변 가능
- 성장가능성 → 가설이 아닌 실사용 데이터(6,227건)로 시장 검증 증명
- 참여역량 → 이미 투자해서 만든 MVP가 의지의 증거

**이 포지셔닝을 사업계획서 전체에서 일관되게 유지한다.**

## 1. 사업 구조 이해

### 1-1. 선발 구조

```
신청 (전체) → STEP1 서류평가 → 1,000개사 선정
                                      ↓
                              AI 활용모델 구축 지원
                              (멘토기업 매칭, 역량 진단)
                                      ↓
                             STEP2 발표평가(오디션)
                                      ↓
                              680개사 사업화 지원
                           (사업화자금 최대 4,000만원)
```

**1차 목표: STEP1 서류평가 통과 (1,000개사)**
**2차 목표: STEP2 발표평가 통과 (680개사) → 사업화자금 수령**

### 1-2. 평가 배점 (STEP1 서류평가 기준)

| 평가 항목 | 배점 | 세부 항목 |
|----------|------|---------|
| AI활용 적합성 | 30점 | 전략 명확성 15점 + 적용 가능성 15점 |
| **성장가능성** | **40점** | 사업모델 개선 20점 + 시장성·경쟁력 20점 |
| 참여역량 | 30점 | 참여 의지 15점 + 계획 구체성 15점 |
| **합계** | **100점** | |

> **성장가능성이 40점으로 최고 배점. 여기서 최대한 점수를 뽑아야 한다.**

## 2. 평가 항목별 점수 최대화 전략

### 2-1. AI활용 적합성 (30점) — "이미 쓰고 있다"

**전략 명확성 (15점)**
- Phase 1~3 로드맵을 구체적 일정(1~2개월 / 3~4개월 / 5~6개월)과 기대효과 수치로 제시
- 각 Phase가 서비스 본질("누가 멈췄는지가 문장의 맥락")을 어떻게 강화하는지 연결
- "AI를 쓰겠다"가 아니라 "현재 OCR·밑줄 감지 → 취향 분석·추천으로 확장"하는 명확한 방향

**적용 가능성 (15점)**
- 핵심 근거: Claude Vision API 프로덕션 6,227건 처리 완료
- 이미 검증된 파이프라인 위에 레이어를 쌓는 것 → 기술적 리스크 없음
- `user_taste_profiles` 테이블 이미 구축 → Phase 2 기반 준비 완료

**작성 포인트:**
```
✓ "Claude Vision API 프로덕션 운영 중 (6,227건 처리 완료)"
✓ "기존 인프라 위에 취향 분석 레이어 추가 — 신규 도입 아님"
✓ "Phase 1 자동 태그 → Phase 2 개인화 피드 → Phase 3 북 큐레이션" 순서 논리적으로 연결
```

---

### 2-2. 성장가능성 (40점) — 여기서 압도해야 한다

**사업모델 개선 (20점)**
- 현재: 무료 서비스 (매출 0원)
- 고도화 후: 3단계 수익화 (어필리에이트 → 프리미엄 구독 → 출판사 B2B)
- 전환율 예측을 수치로 명시 (6개월 MAU 1,000명 → 유료 50명 → 월 245,000원)
- **핵심 논리**: "AI 기능이 추가될수록 유료 전환 가치가 명확해진다" (AI가 비즈니스 모델 개선의 직접 원인)

**시장성·경쟁력 (20점)**
- 시장 크기: BookTok 2,000억 뷰 + #북스타그램 1.2억 건 → 행동은 있고 인프라 없음
- 경쟁 공백: 종이책+소셜+한국어+무료 조합 = 세계에 없음
- 네트워크 효과: 유저가 쌓일수록 각 문장의 "맥락 데이터"가 풍부해짐 → 후발주자 진입 장벽

**작성 포인트:**
```
✓ BookTok 2,000억 뷰 / #북스타그램 1.2억 건 (시장 크기 수치)
✓ 경쟁사 비교표 (명언앱/Readwise 대비 4가지 공백)
✓ 네트워크 효과 설명 (데이터가 쌓일수록 가치 증대)
✓ 수익 전환율 예측 표 (6개월/12개월/18개월)
✓ 한강 노벨문학상 이후 한국 문학 밑줄 공유 폭발 언급 (타이밍)
```

---

### 2-3. 참여역량 (30점) — "이미 했다"

**참여 의지 (15점)**
- 이미 자비로 서비스를 만들고 운영 중 → 의지는 말이 아닌 행동으로 증명
- 6,227건 밑줄 데이터 = 이미 투입한 노력의 결과
- 1인 기업이지만 Claude AI 자동화로 운영 체계 구축 → 지속 가능성 증명

**계획 구체성 (15점)**
- Phase별 구체적 월별 일정 명시
- KPI 수치 (MAU, 밑줄 수, 유료 구독자, 매출) 명확히 제시
- 자금 활용 항목별 금액과 목적 구체화

**작성 포인트:**
```
✓ "현재 운영 중인 서비스의 고도화" (새로 만드는 것이 아님)
✓ KPI 수치 3개 이상 (MAU 1,000명 / 밑줄 30,000개 / 유료 50명)
✓ Phase 1~3 각각 "구체적 개발 항목 + 기간 + 기대효과" 세트로 제시
```

| 금지 표현 | 대체 표현 |

- UI/UX(800만원) → "AI 비즈니스 모델 구현을 위한 사용자 경험 고도화"

### 3-2. 일반 금지 표현

| 금지 | 이유 | 대체 |
|------|------|------|
| "~할 예정입니다" | 불확실성 → 낮은 점수 | "~을 구축합니다" |

## 4. 약점 공략 대응 전략

### 4-1. 매출 0원 (가장 큰 약점)

**대응 논리:**
> "MVP 검증 단계이며, 6,227건의 실사용 데이터가 시장 수요를 증명합니다.
> 매출이 없는 이유는 수익화 기능이 미구현 상태이기 때문이며,
> 본 사업으로 프리미엄 구독 기능을 구현해 6개월 내 첫 수익을 시작합니다."

**작성 원칙**: 매출 0원을 "현재 상태"로 먼저 인정하고, "이유"와 "해결 계획"을 즉시 제시

### 4-2. 1인 기업

**대응 논리:**
> "Claude AI 자동화로 1인 운영 가능한 구조를 구축했습니다.
> OCR·밑줄 감지·책 정보 매칭·취향 집계 모두 자동화.
> 본 사업 완료 후 마케팅 인력 1인 채용으로 성장을 가속화합니다."

### 4-3. 수상·지재권 없음

**대응 논리:**
> "수상·지재권은 없으나, 프로덕션 운영 경험과 기술 지표로 역량을 증명합니다.
> Claude Vision OCR 정확도 95%+, API 처리량 6,227건, 한국어 종이책 특화 파이프라인."

### 4-4. MAU ~50명 (소규모)

**대응 논리:**
> "현재 마케팅 투자 없이 자연 성장한 시드 유저 수입니다.
> 본 사업의 마케팅 자금(800만원)이 투입되면 6개월 MAU 1,000명이 현실적입니다."

## 6. STEP2 발표평가 대비

STEP1 통과 후 발표평가(오디션) 준비사항.

### 6-1. 필수 사항
- **주 대표자 본인 직접 발표** (대리 불가, FAQ 명시)
- 발표 시간 미정 → 5분/10분 버전 모두 준비

### 6-2. 발표 구성 (추천)
```
1. 한 문장 (30초)
   "당신이 멈춘 문장, 누군가도 거기서 멈췄다."

2. 문제 (1분)
   독자들은 이미 인스타에 밑줄 사진을 올린다. 그 문장은 이미지에 갇혀있다.

3. 솔루션 + 데모 (2분)
   사진 한 장 → Claude Vision → 6,227건 처리 완료. 실제 서비스 화면 시연.

4. 왜 우리가 이길 수 있나 (1분)
   종이책+소셜+한국어+무료 = 경쟁자 없음. 네트워크 효과로 선점 효과 극대화.

5. 자금 활용 + KPI (1분)
   Phase 1~3, 6개월 MAU 1,000명, 12개월 유료 400명.
```

### 6-3. 데모 준비
- 실제 서비스 스크린샷 or 영상 준비
- 밑줄 업로드 → 자동 인식 → 피드 등록 전체 플로우
- 6,227건 피드 화면 캡처 (실사용 데이터 증거)
