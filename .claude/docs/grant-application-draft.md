**사업아이템 한줄요약**
독자들이 책을 읽다 멈춘 문장을 Claude AI로 기록하고, 같은 문장에서 멈춘 독자들을 연결하는 소셜 독서 플랫폼 고도화

## [일반 현황]

### 1. 기업 소개

(주)얼리커뮤니케이션은 종이책 독자들의 밑줄 경험을 디지털화하는 소셜 독서 플랫폼 **밑줄(Underline)**을 운영합니다.

독자가 책을 읽다 형광펜·볼펜으로 표시한 문장을 사진 한 장으로 촬영하면, Claude Vision AI가 밑줄 친 문장과 책 정보를 자동으로 인식해 저장합니다. 저장된 문장은 소셜 피드로 공개되어 같은 문장에서 멈춘 다른 독자들과 연결됩니다.

**서비스 핵심 가치**: 편집자가 고른 명언이 아닌, 실제 독자들이 읽다 멈춘 문장들의 집합. 누가 멈췄는지가 그 문장의 맥락이 됩니다.

| 항목 | 내용 |
|------|------|
| 서비스명 | 밑줄 (Underline) |
| 서비스 URL | (Vercel 배포, GitHub 관리) |
| 기술 스택 | Next.js 15, Supabase, Claude Vision API |
| 핵심 AI | Claude Vision API (claude-sonnet-4-6) — OCR + 밑줄 감지 + 책 식별 단일 호출 |
| 운영 현황 | MVP 프로덕션 운영 중 |

---

### 2. 지원 동기

**이미 검증된 Claude AI를 고도화합니다.**

밑줄은 Claude Vision API를 프로덕션에서 이미 운영 중입니다. 사진 한 장으로 밑줄 친 문장과 책 정보를 자동 인식하는 핵심 파이프라인이 6,227건의 실사용 데이터로 검증되었습니다.

이번 지원사업을 통해 현재 OCR·밑줄 감지에 집중된 AI 활용을 **취향 분석·자동 태그·개인화 추천**으로 확장합니다. 이미 구축된 `user_taste_profiles` 테이블(밑줄·좋아요 행동 기반 자동 집계)을 Claude AI가 분석해 "나와 비슷한 독자들의 밑줄"을 추천하는 개인화 엔진을 구현합니다.

**지원 목적 3가지:**
1. AI 자동 태그 생성 → 사용자 입력 부담 70% 감소, 검색 품질 향상
2. 취향 기반 피드 개인화 → 체류시간 향상, 이탈률 감소
3. AI 북 큐레이션 + 어필리에이트 → 첫 수익 실현

**독자들은 이미 인스타그램에 밑줄 사진을 올립니다. 그 행동을 연결할 인프라가 없을 뿐입니다.**

|---|---|---|---|
| 문장 출처 | 편집자 선택 | 전자책 하이라이트 | **실제 독자의 손밑줄** |
| 문장 맥락 | 없음 | 없음 | **누가 멈췄는지 = 맥락** |

핵심 차별점: 문장의 가치는 그것에 멈춘 독자들의 프로필로 결정됩니다. 30년 경력 마케터가 밑줄 친 문장과 대학생이 밑줄 친 문장은 같은 텍스트라도 다른 의미를 가집니다. 이 "문장의 맥락 데이터"는 플랫폼이 성장할수록 후발주자가 따라오기 어려운 데이터 자산이 됩니다.

#### 서비스 상세

**핵심 플로우:**
```
① 사용자가 책 페이지 사진 촬영
② Claude Vision API 단일 호출
   - OCR: 전체 텍스트 추출
   - 밑줄 감지: 형광펜·볼펜·연필·동그라미 등 모든 표시 자동 인식
   - 책 식별: 헤더/푸터 분석 → 카카오 도서 API 매칭
③ 밑줄 문장 + 책 정보 확인 (수동 수정 가능)
④ 저장 → 소셜 피드 공개
⑤ "같은 문장에서 멈춘 N명" 연결
```

**주요 기능:**
- 소셜 피드 (좋아요, 댓글)
- 책 단위 밑줄 모음 + 페이지 히트맵
- 공유 카드 생성 (6종 테마, 인스타그램 최적화)
- 취향 프로파일 (행동 기반 자동 분석)

### 7. AI 활용 아이템 소개

**"독자가 멈춘 문장에서 그 사람의 취향을 읽는다"**

현재 밑줄은 Claude Vision으로 밑줄 문장을 기록합니다. 이번 사업으로 **기록된 문장들을 Claude가 분석해 독자의 취향을 파악하고, 그 취향을 기반으로 새로운 문장과 책을 추천하는** 지능형 독서 플랫폼으로 고도화합니다.

**고도화 전후 비교:**

| | 현재 (고도화 전) | 목표 (고도화 후) |
|---|---|---|
| AI 역할 | 사진 분석 (OCR + 밑줄 감지) | 사진 분석 + 취향 분석 + 개인화 추천 |
| 태그 | 사용자 수동 입력 | Claude 자동 생성·추천 |
| 피드 | 시간순 단순 정렬 | 취향 기반 개인화 |
| 책 추천 | 없음 | AI 취향 분석 → 다음 책 3권 추천 |
| 수익화 | 없음 | 어필리에이트 + 프리미엄 구독 |

### 8. AI활용 모델 구축 계획

**Phase 1 (1~2개월): AI 자동 태그 생성**

- **문제**: 사용자가 밑줄 저장 시 태그를 수동 입력 → 입력 부담이 완료율 저하 원인
- **해결**: 밑줄 문장을 Claude API가 분석 → 적합한 태그(#성장, #위로, #인생 등) 3~5개 자동 추천
- **구현**: `/api/tags/suggest` 엔드포인트 신설, Claude API 텍스트 분석 적용
- **기대효과**: 태그 입력 시간 70% 단축, 태그 일관성 향상 → 검색 품질 개선

**Phase 2 (3~4개월): 취향 기반 피드 개인화**

- **문제**: 모든 사용자에게 동일한 시간순 피드 → 관련성 낮은 콘텐츠로 이탈 발생
- **기반**: `user_taste_profiles` 테이블 이미 구축 완료 (밑줄·좋아요 행동 기반 자동 집계)
- **해결**: 축적된 취향 데이터를 Claude API가 분석 → "나와 비슷한 독자들의 밑줄" 피드 우선 노출
- **구현**: 취향 벡터 유사도 계산 + Claude API 맥락 분석 결합
- **기대효과**: 관련성 높은 피드 → 체류시간 향상, 이탈률 감소

**Phase 3 (5~6개월): AI 북 큐레이션 & 수익화 연결**

- **해결**: 사용자 밑줄 히스토리를 Claude API가 분석 → 다음 읽을 책 3권 추천 + 도서 구매 링크
- **구현**: Claude API 취향 분석 + 카카오 도서 API 연동
- **기대효과**: 도서 어필리에이트 수익 시작, 프리미엄 구독 전환 유도
