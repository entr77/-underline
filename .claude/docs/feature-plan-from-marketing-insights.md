### 1. 감정 태그 자동 추천

**출처 인사이트**: 바이럴 공식 — 각성 수준 높은 감정(경이·그리움)이 슬픔보다 바이럴 강함

**유저 스토리**: "나는 밑줄을 저장할 때 이 문장이 나에게 어떤 감정인지 빠르게 고르고 싶다. 같은 감정으로 이 문장을 느낀 다른 독자를 보고 싶다."

**핵심 동작**:
- `/new` 저장 단계에서 감정 이모지 칩 선택 UI 추가 (최대 1개, 선택 선택사항)
- 감정 셋: 경이 / 그리움 / 쓸쓸함 / 설렘 / 위로 / 공감 / 충격 (바이럴 강도 순)
- 기존 태그 컬럼에 `emotion:경이` 형태로 저장 (별도 컬럼 불필요)
- 피드 필터에 감정 탭 추가 → 기존 FeedFilter 컴포넌트 확장

**연결 지점**: `/new` 멀티스텝 플로우, 기존 FeedFilter

**예상 효과**: 감정 필터로 피드 체류 시간 증가 / "경이" 태그 밑줄 저장·DM 공유 증가

**개발 난이도**: 낮음

### 2. "같은 문장에서 멈춘 독자" 연결 강화

**출처 인사이트**: "여기서 같이 멈췄어요"가 핵심 감정 훅인데 충분히 활용 안 됨

**유저 스토리**: "나는 나와 같은 밑줄을 그은 독자가 어떤 다른 문장에도 밑줄을 그었는지 보고 싶다. 취향이 맞는 사람을 발견하고 싶다."

**핵심 동작**:
- 밑줄 상세 페이지의 독자 아바타 탭 → 해당 독자의 미니 카드 슬라이드업
- 미니 카드: 아바타 + 닉네임 + "이 독자의 다른 밑줄 3개 미리보기" (같은 책 우선)
- 프로필 전체 보기 링크 → `/profile/[username]`

**연결 지점**: 기존 밑줄 상세 "여기서 같이 멈췄어요" 섹션, 프로필 페이지

**예상 효과**: 독자 간 연결 경험 강화 → 팔로우 전환율 상승 (팔로우 기능 구현과 시너지)

**개발 난이도**: 낮음

---

### 3. 책 완독 배지 & "다 읽었어요" 선언

**출처 인사이트**: 소셜 공유는 완독 후 며칠 뒤에 일어남 / 완독자일수록 밑줄 공유율 높음

**유저 스토리**: "나는 책을 다 읽은 뒤 이 책에서 밑줄 그은 문장들을 한꺼번에 돌아보고 싶다. 완독했다는 사실을 기록으로 남기고 싶다."

**핵심 동작**:
- `/book/[id]` 하단에 "이 책 다 읽었어요" 버튼 추가
- `user_readings` 테이블에 완독 일자 저장 (DB 마이그레이션 필요)
- 완독 선언 후 해당 책에서 내 밑줄 전체 타임라인 화면 진입
- 프로필 페이지에 완독 책 수 배지 표시

**연결 지점**: `/book/[id]`, 마이페이지 프로필 섹션

**예상 효과**: 완독 직후 공유 이미지 카드 생성 전환율 상승 / 프로필 재방문 동기

**개발 난이도**: 낮음 (DB 테이블 1개 추가, UI 버튼)

### 4. 에세이 장르 전용 탐색 뷰

**출처 인사이트**: 에세이 독자 밑줄 복귀율 β=0.53 (소설 0.00) — 에세이가 밑줄 행동 압도적 지배

**유저 스토리**: "나는 에세이·산문 책에서 나온 밑줄만 모아서 보고 싶다. 소설 스포일러 없이 좋은 문장만 발견하고 싶다."

**핵심 동작**:
- 카카오 API `category_name`을 `books` 테이블에 저장 (저장 여부 선행 확인 필요)
- `/books` 상단에 장르 필터 탭: 전체 / 에세이·산문 / 소설 / 인문·사회
- 에세이 특화 큐레이션 문구: "오늘의 에세이 문장" (밑줄 수 + 좋아요 기준 1위)

**연결 지점**: 기존 `/books` 책 피드, 카카오 API 저장 로직

**예상 효과**: 에세이 독자(가장 충성도 높은 세그먼트) 직접 공략

**개발 난이도**: 낮음~중간 (카테고리 데이터 저장 여부에 따라)

---

### 5. 여러 밑줄 묶음 공유 (캐러셀)

**출처 인사이트**: 인스타그램 캐러셀이 단일 이미지보다 저장률 22% 높음 / 완독 후 선별 공유 패턴

**유저 스토리**: "나는 이 책에서 밑줄 그은 문장 여러 개를 한 번에 골라 하나의 인스타그램 게시물로 공유하고 싶다."

**핵심 동작**:
- `/book/[id]` 또는 `/profile/[username]`에서 멀티 선택 모드 진입 (롱탭 or 버튼)
- 선택한 밑줄 2~6개를 기존 OG 카드 API 반복 호출로 PNG 생성
- ZIP으로 묶어 다운로드 / Web Share API로 복수 파일 공유

**연결 지점**: 기존 `/api/og/underline/[id]` API 재활용

**예상 효과**: 저장률 22% 상승 / 책 단위 공유 생성 → `/book/[id]` 유입 경로 다변화

**개발 난이도**: 중간
