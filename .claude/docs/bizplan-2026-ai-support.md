## 2. 서비스 본질 & 핵심 메시지

### 서비스 본질 (모든 메시지의 뿌리)

> **당신이 멈춘 문장, 누군가도 거기서 멈췄다.**

독자가 책을 읽다 멈추는 순간에는 이유가 있다.
편집자가 고른 명언이 아니라, 실제 독자들이 멈춘 문장 — **누가 멈췄는지가 그 문장의 맥락이 된다.**

이게 굿리즈, 명언 앱, Readwise와 다른 지점이다.
그들은 문장을 모은다. 밑줄은 문장에 독자의 시선을 붙인다.

### 사업계획서 전체에서 반복할 메시지 3가지

1. **"이미 쓰고 있다"** — Claude Vision API 프로덕션 운영 중. 리스크 없는 확장.
2. **"아무도 안 한다"** — 종이책+소셜+한국어+무료 조합. 특히 "독자의 시선이 문장의 맥락이 되는" 구조는 세상에 없다.
3. **"시장이 이미 존재한다"** — 독자들은 이미 인스타에 밑줄 사진을 올린다. 행동은 있고 인프라가 없다.

| 단계 | 수익원 | 대상 | 시점 |

> **인용 서비스 핵심**: 크리에이터·방송작가·광고기획자가 "내 청중과 비슷한 독자들이 반응한 문장"을 찾을 때, 단순 명언 앱이 아닌 **독자 프로파일이 붙은 인용문**이 필요하다. 출처(책명·저자·페이지) + 어떤 독자들이 멈췄는지 맥락까지 제공. RAG 기반으로 상황(직업·청중·목적)을 입력하면 최적 인용문을 자동 추천.

> **출판사 B2B**: "이 책에서 독자들이 가장 많이 멈춘 페이지·문장 TOP10 + 멈춘 독자의 성향 분포" 리포트. AI가 책 전체 밑줄을 분석해 저자 대시보드로 제공.

## 5. 마케팅 전략

### 핵심 채널 (우선순위 순)

| 채널 | 방식 | 왜 효과적인가 |
|------|------|--------------|
| **저자** | 저자에게 "내 책 밑줄 데이터" 무료 제공 → SNS 홍보 유도 | 저자가 공유하면 팬덤 자동 유입 + 출판사 B2B 연결 |
| **북클럽·독서 모임** | 모임 내 무료 체험 → 밑줄 공유 → 피드 형성 | 이미 책 얘기하는 사람 집합소. 전환율 최고 |
| **북스타그래머** | 사용 경험 기반 콘텐츠 협업 (광고 X) | "써봤더니 좋더라" 진정성이 팔로워 신뢰로 이어짐 |
| **오픈카톡방 독서 모임** | 서비스 소개 + 밑줄 카드 예시 공유 | 타겟 집결, 비용 0 |
| **어필리에이트** | 밑줄 상세 → "이 책 구매하기" (쿠팡, 알라딘) | 책에 관심 있는 순간에 노출 |

### 마케팅 예산 800만원

| 항목 | 금액 | 내용 |
|------|------|------|
| 저자 파트너십 | 200만원 | 저자 10~20명 대시보드 무료 제공 + 공유 요청 |
| 북스타그래머 체험 협업 | 250만원 | 팔로워 1만~5만 계정 5명, 체험비 |
| 독서 모임 파트너십 | 150만원 | 주요 북클럽 3~5곳 이벤트 비용 |
| 콘텐츠 제작 | 100만원 | 카드뉴스, 릴스, 사용 가이드 |
| SNS 광고 | 100만원 | 인스타 북스타그램 관심사 타겟 |

### 바이럴 루프

```
독자 사진 촬영 → Claude AI → 밑줄 + 책 자동 인식 → 저장
        ↓
"같은 문장에서 멈춘 사람들" 알림 → 공명 경험
        ↓
공유 카드 생성 (6종 테마, "당신이 멈춘 문장" 카피 포함)
        ↓
인스타 스토리/피드 업로드 → 앱 링크 자동 포함
        ↓
팔로워 방문 → "나도 이 문장에서 멈췄는데" → 신규 가입 (반복)
```

> 바이럴의 핵심은 카드 디자인이 아니라 **"나도 여기서 멈췄다"는 공명**이다.
> 같은 문장에서 멈춘 사람이 이미 있다는 사실이 클릭을 만든다.

## 6. AI 전략 — Claude + 자체 RAG 이중 구조

### 핵심 전략: 데이터가 쌓일수록 Claude 의존도를 낮춘다

현재는 누적 데이터가 부족해 Claude API로 모든 분석을 처리한다.
밑줄·책·유저 데이터가 충분히 쌓이면 **자체 RAG(Retrieval-Augmented Generation)** 로 대체해 API 비용을 낮추고 응답 품질을 높인다.

| 단계 | AI 구성 | 시점 |
|------|--------|------|
| 현재 (데이터 축적기) | Claude API 전담 — OCR, 분석, 추천 모두 처리 | 운영 중 |
| Phase 2 이후 (RAG 전환기) | 자체 벡터 DB(pgvector) + Claude 혼합 — 반복 쿼리는 RAG, 창의적 분석은 Claude | 3개월 이후 |
| 장기 (자립기) | 자체 RAG 주도 — Claude는 보완 레이어로 축소 | 12개월 이후 |

> **RAG 구성 기반**: Supabase에 이미 축적 중인 밑줄 텍스트 + 책 메타데이터 + 유저 행동 데이터를 pgvector로 벡터화. 유사 문장·유사 독자 검색을 외부 API 없이 자체 처리.

### AI 기능 전체 맵

| 기능 | 현재 | 고도화 후 | AI 방식 |
|------|------|---------|--------|
| 책 사진 → 도서 추출 | ✅ Claude Vision | 유지 | Claude Vision |
| 밑줄 문장 OCR | ✅ Claude Vision | 유지 | Claude Vision |
| 밑줄 감지 (형광펜·볼펜) | ✅ Claude Vision | 유지 | Claude Vision |
| 장르·태그 자동 분류 | ✗ 수동 | **자동화** | Claude → RAG 전환 |
| 유저 성향 분석 (직업·독서 패턴) | △ 기초 집계 | **심층 분석** | Claude 분석 |
| 동일 밑줄 클러스터링 | ✗ | **자동화** | pgvector 유사도 검색 |
| 밑줄별 공명 독자 프로파일링 | △ 기초 | **자동화** | RAG + 통계 |
| 책 요약표 생성 (책별 밑줄 분석) | ✗ | **자동화** | Claude 분석 → RAG |
| 책 추천 | ✗ | **자동화** | RAG + Claude |
| 상황별 인용문 추천 | ✗ | **자동화** | RAG 검색 |

### AI 도입 로드맵 (Phase별)

| Phase | 기간 | 핵심 개발 내용 | 기대효과 |
|-------|------|-------------|---------|
| **Phase 1** | 1~2개월 | **AI 자동 분석 엔진** — ① 밑줄 문장 → 장르·태그 자동 분류 (Claude) ② 유저 직업·독서 성향 심층 프로파일링 ③ 동일 밑줄 클러스터링 (pgvector) | 태그 입력 70% 단축, 유저 프로파일 자동화 |
| **Phase 2** | 3~4개월 | **RAG 기반 개인화 + 책 요약표** — ① pgvector 자체 벡터 DB 구축 ② 취향 기반 피드 개인화 ("나와 비슷한 독자의 밑줄") ③ 책별 밑줄 자동 요약표 생성 (저자/출판사 대시보드 기반) | Claude 비용 30%↓, 체류시간 향상 |
| **Phase 3** | 5~6개월 | **추천 엔진 + 수익화 연결** — ① AI 책 추천 (밑줄 히스토리 → 다음 책 3권) ② 상황별 인용문 추천 (직업·상황 입력 → 적합한 밑줄 검색) ③ 인용 서비스·어필리에이트 수익 연결 | CPS 수익 시작, 프리미엄 전환 유도 |
