## 1. 저작권 정책 가이드라인 (밑줄 특화)

### 저작권법 제28조 "인용" 조항: 밑줄에 적용 가능한 안전한 범위

저작권법 제28조는 공표된 저작물을 보도·비평·교육·연구 등을 위해 정당한 범위 안에서 공정한 관행에 합치되게 인용할 수 있다고 규정한다. 밑줄 서비스가 이 조항을 근거로 방어하려면 다음 요건을 충족해야 한다.

**인용 정당성 요건 (판례 기준)**

1. **주종 관계**: 인용된 문장이 서비스의 주가 아니라 종이어야 한다. 밑줄의 경우 유저의 감상·맥락·커뮤니티 연결이 주이고, 인용된 책 문장이 종이라는 구조로 설계되어야 한다.
2. **출처 명시**: 책 제목·저자·출판사를 반드시 함께 표시해야 한다 (저작권법 제37조).
3. **비상업적 맥락**: 순수 광고·판매 목적이 아닌 독서 커뮤니티·감상 공유 목적임을 명확히 한다.
4. **공표된 저작물**: 이미 출판된 책만 대상이며 미공표 원고는 인용 불가.

### 안전한 문장 저장 범위 기준

| 기준 | 안전 범위 | 주의 범위 | 위험 범위 |
|------|-----------|-----------|-----------|
| 글자 수 (한글 기준) | 200자 이내 (1~2문장) | 200~500자 (단락 수준) | 500자 초과 |
| 문단 수 | 1문단 이내 | 2~3문단 연속 | 4문단 이상 |
| 책 전체 대비 비율 | 0.1% 미만 | 0.1~1% | 1% 초과 |
| 연속 페이지 저장 | 1페이지 이내 | 2~3페이지 | 4페이지 이상 |

**운영 원칙**: OCR 추출 후 사용자가 확인·편집하는 구조로 유지하여 서비스가 자동 대량 복제하는 것이 아님을 명확히 한다. 사용자가 직접 밑줄 친 문장 1~3개를 선택·저장하는 행위가 핵심 플로우다.

### 위험 시나리오

**시나리오 1: 전체 챕터 OCR 저장**
- 위험도: 높음 (High)
- 근거: 복제권 침해 (저작권법 제16조), 인용 조항 적용 불가
- 대응: OCR 추출 후 밑줄 감지 문장만 저장되도록 기술적 제한 유지. 현재 아키텍처(`highlight-analyzer.ts`)가 밑줄 문장만 추출하는 방식으로 설계되어 있어 이 원칙에 부합함.

**시나리오 2: 동일 책에서 대량 문장 수집**
- 위험도: 중간~높음 (Medium-High)
- 근거: 개별 문장은 인용 범위이나, 집적(aggregation) 시 사실상 책 내용 전체를 재구성할 수 있어 실질적 복제에 해당할 수 있음.
- 대응: 동일 책 동일 유저 기준 월 30개 이상 저장 시 모니터링 플래그 설정 검토.

**시나리오 3: 밑줄 카드 이미지로 SNS 배포**
- 위험도: 낮음~중간 (Low-Medium)
- 근거: 출처가 명시된 짧은 인용은 공정 이용 범위. 단, 저작물이 카드 이미지의 주인공이 되면 위험도 상승.
- 대응: 카드에 반드시 책 제목·저자를 표시하고, 출판사 로고/ISBN 추가 표시를 권장.

### 출처 표시 의무

저작권법 제37조(출처의 명시)에 따라 인용 시 출처를 명시해야 한다. 밑줄 서비스의 모든 저장 문장에는 다음이 필수로 포함되어야 한다.

- **책 제목** (필수)
- **저자명** (필수)
- **출판사** (필수)
- ISBN (권장 — 카카오 도서 API에서 자동 수집 가능)

현재 DB 스키마(`books` 테이블)에 이 정보가 포함되어 있으며, `underlines` 테이블의 `book_id` 외래 키로 연결되어 있다. 이 구조가 출처 표시 의무 이행의 기술적 근거가 된다.

### UGC 플랫폼 면책 조항 (정보통신망법 제44조의2)

정보통신망법 제44조의2(정보의 삭제요청 등) 및 OSP(Online Service Provider) 면책 법리에 따라 밑줄은 다음 조건을 갖춰야 한다.

1. **저작권 침해 신고 접수 채널 운영**: 이메일(예: copyright@underline.kr) 또는 인앱 신고 기능 필수
2. **통보 및 삭제(Notice and Takedown)**: 신고 접수 후 영업일 3일 이내 검토, 침해 확인 시 즉시 비공개 처리
3. **반복 침해자 계정 조치**: 3회 이상 침해 확인 시 계정 정지 정책 수립
4. **서비스 내 저작권 정책 게시**: 이용약관 및 저작권 정책 페이지 공개 필수

### 저작권 침해 신고 접수 시 처리 프로세스

```
1. 신고 접수
   - 채널: 이메일 / 인앱 신고 버튼
   - 접수 확인 메일 발송 (영업일 1일 이내)

2. 검토 (영업일 3일 이내)
   - 신고 내용 확인: 저작물 특정, 침해 위치, 권리자 증명
   - 해당 밑줄의 문장 길이·출처 표시 여부 검토

3. 조치
   - 침해 확인: 해당 밑줄 비공개 처리 → 유저 통지
   - 침해 불명확: 신고자에게 추가 증빙 요청
   - 침해 아님: 신고자에게 결과 통보

4. 반복 침해 처리
   - 동일 유저 3회 확인 시: 계정 경고
   - 5회 이상: 계정 정지 (이용약관 근거 필요)

5. 기록 보관
   - 신고 접수~처리 이력 6개월 보관
```

## 2. PIPA (개인정보보호법 2023) 준수 체크리스트

### 수집 항목별 법적 근거

| 수집 항목 | 법적 근거 | 수집 목적 | 보존 기간 | 위험도 |
|-----------|-----------|-----------|-----------|--------|
| 이메일 | 계약 이행 (법 제15조 제1항 제4호) | 계정 인증, 서비스 이용 | 탈퇴 후 30일 | 낮음 |
| 닉네임 | 계약 이행 | 프로필 표시 | 탈퇴 후 30일 | 낮음 |
| 프로필 사진 | 정보주체 동의 (법 제15조 제1항 제1호) | 프로필 표시 | 탈퇴 후 30일 | 낮음 |
| 업로드 이미지 | 계약 이행 | OCR 처리 및 밑줄 저장 | 밑줄 삭제 시 | 중간 |
| 독서 기록 (밑줄) | 계약 이행 | 서비스 핵심 기능 | 탈퇴 후 30일 | 낮음 |
| 좋아요 기록 | 계약 이행 | 소셜 기능 | 탈퇴 후 즉시 | 낮음 |
| EXIF 메타데이터 | 수집 불가 | — | 즉시 제거 필수 | 높음 |
| Google OAuth 토큰 | 계약 이행 | 소셜 로그인 | 세션 만료 시 | 낮음 |
| 접속 로그 (IP, UA) | 정당한 이익 (법 제15조 제1항 제6호) | 보안, 어뷰징 방지 | 3개월 | 낮음 |

### EXIF 메타데이터 반드시 제거해야 하는 이유

카메라/스마트폰으로 찍은 사진의 EXIF 데이터에는 GPS 좌표(위도·경도)가 포함될 수 있다. 이는 개인정보보호법상 위치정보(개인정보 처리방침 필수 고지 항목)에 해당하며, 위치정보의 보호 및 이용 등에 관한 법률(위치정보법) 적용 대상이 된다.

- **조치 방법**: 이미지 업로드 시 서버 사이드에서 EXIF 전체 제거 후 저장 (sharp 라이브러리 등 활용)
- **현재 상태 확인 필요**: `/api/vision/analyze` 라우트에서 이미지 저장 전 EXIF 제거 여부 점검 (Medium Risk)

### 보존 기간 정책

**이용 중**
- 모든 개인정보: 서비스 이용 기간 동안 보존
- 업로드 원본 이미지: OCR 처리 완료 후 Supabase Storage에 유지 (유저가 삭제하기 전까지)

**탈퇴 후**
- 계정 정보 (이메일, 닉네임): 30일 후 완전 삭제
  - 근거: 부정이용 방지 목적 단기 보존 허용 (법 제21조)
- 공개 밑줄: 탈퇴 후 익명 처리(작성자 NULL 처리) 또는 삭제 — 서비스 정책 결정 필요
- 이미지 파일: 탈퇴 후 30일 이내 Storage에서 삭제
- 접속 로그: 3개월 후 삭제

**법적 의무 보존**
- 전자상거래 등에서의 소비자보호에 관한 법률에 따라 거래 기록이 발생하면 5년 보존 (현재 무료 서비스이므로 해당 없음, 유료화 시 재검토)

### 개인정보처리방침 필수 기재 항목

개인정보보호법 제30조 및 표준 개인정보보호지침에 따라 다음 항목을 처리방침에 기재해야 한다.

1. 개인정보의 처리 목적
2. 개인정보의 처리 및 보유 기간
3. 개인정보의 제3자 제공 현황 (Google OAuth, Supabase, Vercel)
4. 개인정보처리의 위탁 현황 (Google Cloud Vision API — 이미지 처리 위탁)
5. 정보주체의 권리·의무 및 행사 방법 (열람, 정정, 삭제, 처리정지 요청)
6. 처리하는 개인정보의 항목
7. 개인정보의 파기 절차 및 방법
8. 개인정보의 안전성 확보 조치 (Supabase RLS, HTTPS, 암호화)
9. 개인정보 보호책임자 성명 및 연락처
10. 처리방침 변경 고지 방법

## 3. 외부 API 이용약관 준수 검토

### 카카오 도서 API

**관련 약관**: 카카오 개발자 서비스 이용약관 및 카카오 데이터 정책

| 항목 | 현황 | 준수 여부 | 조치 사항 |
|------|------|-----------|-----------|
| 상업적 사용 | 무료 서비스 운영 중 | 조건부 허용 | 유료화 전환 시 재검토 필수 |
| 검색 결과 캐싱 | 허용 (단, 기간 제한 있음) | 현재 캐싱 여부 확인 필요 | 캐시 유효기간 24시간 이내 권장 |
| API 호출 서버 위치 | 서버에서만 호출 (`/api/books/search`) | 적합 | 클라이언트 직접 노출 금지 유지 |
| 검색 결과 수정 표시 | 카카오 출처 표시 의무 | 확인 필요 | 책 정보 표시 시 "카카오 제공" 또는 로고 표시 검토 |
| 크롤링/대량 수집 | 금지 | 적합 | 현재 단건 검색만 사용 |

**주의사항**: 카카오 API를 통해 가져온 책 커버 이미지 URL을 직접 DB에 저장하는 경우, 해당 이미지에 대한 저작권이 출판사/카카오에 있으므로 이미지를 서비스 내에 재복제(Supabase Storage 저장)하지 않고 원본 URL만 참조하는 방식이 안전하다.

### Google Cloud Vision API

**관련 약관**: Google Cloud 서비스 이용약관 및 Google Cloud API 이용 정책

| 항목 | 현황 | 준수 여부 | 조치 사항 |
|------|------|-----------|-----------|
| 이미지 데이터 처리 | Google 서버로 이미지 전송 | 개인정보 위탁 처리에 해당 | 처리방침에 위탁 현황 기재 필수 |
| 데이터 보존 | Google은 API 처리 후 데이터를 보존하지 않음 (Google 정책 확인 필요) | 확인 필요 | DPA 체결 여부 확인 |
| 상업적 사용 | 허용 | 적합 | 유료 플랜 사용 시 문제 없음 |
| 민감 정보 포함 이미지 | 처리 시 주의 | 잠재적 리스크 | 사용자 얼굴이 포함된 사진 처리 시 생체정보 해당 가능 |

**중요**: 이미지에 사람 얼굴이 포함된 경우 생체정보(개인정보보호법 제23조 민감정보)에 해당할 수 있다. 현재 서비스는 책 페이지 OCR 목적이므로 얼굴 감지를 명시적으로 사용하지 않으나, Vision API가 부수적으로 처리할 가능성을 배제할 수 없다. 처리방침에 명시 권장.

### Supabase

**관련 약관**: Supabase 이용약관 및 DPA(Data Processing Agreement)

| 항목 | 현황 | 준수 여부 | 조치 사항 |
|------|------|-----------|-----------|
| DPA 체결 | 미확인 | PIPA/GDPR 준수 위해 필요 | Supabase DPA 서명 필요 (무료 플랜도 가능) |
| 데이터 저장 위치 | 프로젝트 설정에 따라 다름 | 한국 리전 없음 | AWS ap-northeast-1(도쿄) 또는 ap-southeast-1(싱가포르) 선택 권장 |
| 데이터 암호화 | 전송 중 TLS, 저장 시 AES-256 | 적합 | — |
| 데이터 소유권 | 서비스 운영자 소유 | 적합 | Supabase는 데이터 처리자, 밑줄은 데이터 처리자 |
| 서비스 중단 시 데이터 반출 | pg_dump 등으로 가능 | 적합 | 주기적 백업 정책 수립 권장 |

**DPA 체결 필요성**: PIPA 제26조(개인정보의 처리 위탁)에 따라 Supabase는 개인정보 처리 수탁자이므로 위탁 계약서 체결이 법적 의무다. Supabase의 DPA는 [Supabase 공식 문서](https://supabase.com/privacy)에서 확인 및 서명 가능.

### Vercel

**관련 약관**: Vercel 이용약관 및 DPA

| 항목 | 현황 | 준수 여부 | 조치 사항 |
|------|------|-----------|-----------|
| 서버리스 환경 데이터 처리 | Route Handler에서 임시 처리 | 일시적 처리이므로 위험도 낮음 | Edge Runtime 사용 시 데이터 위치 확인 필요 |
| 로그 보존 | Vercel 함수 로그는 기본 1~7일 | 개인정보 포함 로그 주의 | 로그에 개인정보 출력 금지 |
| DPA 체결 | Vercel DPA 제공 | GDPR/PIPA 준수 위해 체결 권장 | Vercel 대시보드 → Legal에서 DPA 확인 |
| 데이터 전송 국가 | 미국 (기본), 엣지는 전 세계 | 국외 이전 해당 | 처리방침에 국외 이전 현황 기재 필수 |

## 4. 이용약관 핵심 조항 (밑줄 서비스용)

### UGC 라이선스: 유저 밑줄에 대한 서비스 표시 권한

유저가 서비스에 저장하는 밑줄 문장은 두 가지 저작권 관계가 중첩된다.

1. **원저작물(책 문장)에 대한 권리**: 저자/출판사에 귀속. 서비스는 인용 플랫폼으로서 원저작물 사용 권한을 주장하지 않는다.
2. **유저의 편집저작물 권리**: 유저가 어떤 문장에 밑줄을 그을지 선택하는 행위에 창작성이 인정될 수 있다(편집저작물, 저작권법 제6조).

**약관에 명시해야 할 UGC 라이선스 조항 (예시)**:

```
회원은 서비스에 밑줄(인용 문장, 감상, 사진 등)을 등록함으로써 회사에 대해
해당 콘텐츠를 서비스 운영, 홍보, 기능 개선 목적으로 사용·복제·수정·배포할 수 있는
전 세계적, 비독점적, 무상의, 재라이선스 가능한 권리를 부여합니다.
단, 원저작물(책 문장)에 대한 권리는 원저작자에게 귀속되며,
회사는 이를 서비스 운영 목적 이외에 사용하지 않습니다.
```

**공개/비공개 선택 기능 관련**: 현재 구현된 공개/비공개 선택 기능(PRD P0 완료)과 연동하여, 비공개 밑줄에 대해서는 서비스 홍보 목적 사용을 제외한다는 조항 추가 권장.

### 계정 제재 기준: 저작권 침해 반복 처리

| 위반 횟수 | 조치 | 유예 기간 |
|-----------|------|-----------|
| 1~2회 | 해당 밑줄 비공개 처리 + 경고 메일 | — |
| 3~4회 | 계정 7일 이용 제한 | 이의신청 3일 이내 |
| 5회 이상 | 계정 영구 정지 | 이의신청 7일 이내 |

이의신청 채널 운영 의무: 이용약관 제재 조항과 함께 이의신청 방법(이메일, 기간)을 명시해야 정보통신망법 준수.

### 분쟁 해결: 준거법, 관할법원

```
본 약관은 대한민국 법률에 따라 해석되고 적용됩니다.
서비스 이용과 관련하여 발생하는 분쟁에 대해서는 민사소송법상
관할 법원을 제1심 법원으로 합니다. (서울중앙지방법원 권장)
소비자 분쟁의 경우 소비자 거주지 관할 법원에서 제기 가능합니다.
```

**국외 유저 대응**: 서비스가 해외 유저를 유치할 경우 GDPR 적용 대상이 될 수 있으며, EU 유저에 대해서는 대한민국 법원 전속 관할 조항이 효력이 없을 수 있다.

## 5. 신규 기능 법무 사전 검토 기준

### Legal 검토가 필요한 기능 체크리스트

다음 중 하나라도 해당하면 개발 전 Legal 에이전트 검토가 필요하다.

- [ ] 유저 데이터를 외부 서비스로 전송하는 기능
- [ ] 유저 행동 데이터를 분석·프로파일링하는 기능
- [ ] 제3자 개인정보를 수집하거나 처리하는 기능
- [ ] 광고성 정보(푸시, 이메일, SMS) 발송 기능
- [ ] 결제, 구독, 유료화 기능
- [ ] 미성년자 식별 또는 관련 콘텐츠 노출 기능
- [ ] 위치정보 수집 기능
- [ ] 생체정보(얼굴, 지문 등) 처리 기능
- [ ] 다른 유저의 데이터를 제3자에게 제공하는 기능
- [ ] 저작물을 자동으로 대량 수집하는 기능

### 책 추천 알고리즘 (프로파일링 규제)

**법적 리스크**: 개인정보보호법 제37조의2(자동화된 결정에 대한 정보주체의 권리)에 따라, 자동화된 결정이 정보주체에게 중요한 영향을 미치는 경우 정보주체는 이에 대한 설명 요구권과 거부권을 가진다.

독서 취향 프로파일링 기반 책 추천은 "중요한 영향"에 해당할 가능성은 낮으나, 다음을 준수해야 한다.

- 추천 로직이 자동화됨을 이용자에게 고지
- 추천 거부(개인화 끄기) 옵션 제공
- 프로파일링에 사용되는 데이터 항목을 처리방침에 명시

### 알림 서비스 (광고성 정보 규제)

**법적 리스크**: 정보통신망법 제50조(광고성 정보 전송 제한)에 따라 광고성 정보는 반드시 명시적 수신 동의를 받아야 한다.

| 알림 유형 | 법적 분류 | 동의 필요 여부 |
|-----------|-----------|----------------|
| 좋아요 알림 | 서비스 알림 (비광고) | 불필요 (서비스 이용 계약 범위) |
| 팔로워 알림 | 서비스 알림 (비광고) | 불필요 |
| 신규 도서 추천 | 광고성 정보 | 별도 수신 동의 필수 |
| 마케팅 이메일 | 광고성 정보 | 별도 수신 동의 필수 + 수신거부 링크 필수 |
| 앱 푸시 (마케팅) | 광고성 정보 | 별도 수신 동의 필수 |

**구현 요건**: 마케팅 알림 발송 전 이용자별 수신 동의 여부를 DB에 기록하고, 수신 거부 처리 이력을 6개월간 보존해야 한다.

### 독서 모임 (제3자 개인정보 공유)

향후 독서 모임 기능 구현 시 주요 법적 리스크:

- **제3자 제공 동의**: 모임 참여자들 간의 프로필(이메일, 닉네임, 사진) 공유는 개인정보의 제3자 제공에 해당 → 별도 동의 필요
- **아동·청소년 보호**: 모임 참여자 연령 확인 체계 없을 경우 만 14세 미만 개인정보 수집 리스크
- **오픈 모임 vs 비공개 모임**: 공개 모임에 노출되는 정보 범위를 이용자가 제어할 수 있어야 함
