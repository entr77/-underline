## 1. 밑줄 서비스 그로스 모델

### 핵심 바이럴 루프

```
[사용자 책 읽기]
      ↓
[사진 찍어 OCR로 밑줄 저장] ← 행동 장벽 최소화 (사진 1장)
      ↓
[공유 이미지 카드 생성] ← 카드 퀄리티가 공유 동기 결정
      ↓
[인스타·틱톡·X 공유] ← 타 독자의 눈에 닿음
      ↓
[신규 유저 랜딩] ← "나도 이런 카드 만들고 싶다" + "같은 책 독자 발견"
      ↓
[가입 → 첫 밑줄 저장] ← 활성화 완료
      ↓
[다시 공유] ← 루프 반복
```

루프의 약한 고리: 공유 카드를 봤을 때 앱 진입 경로가 불명확하면 루프 단절. 공유 카드에 반드시 watermark(앱명 + URL) 포함, 상세 페이지 OG 이미지 최적화.

### 활성화 North Star 지표

| 지표 | 정의 | 목표치 |
|------|------|--------|
| **Time-to-First-Underline** | 가입 완료 ~ 첫 밑줄 저장 완료까지 경과 시간 | **24시간 이내 70%** |
| **Activation Rate** | 가입 7일 이내 밑줄 1개 이상 저장한 유저 비율 | **50% 이상** |

**근거**: 에세이 독자는 텍스트힙 행동 패턴상 즉각 저장 성향이 높다. 가입 후 24시간 내 첫 경험이 없으면 망각 곡선에 의해 72시간 이후 복귀율이 급격히 하락한다. 목표 24시간 이내 70%는 모바일 앱 업계 평균(~40%)보다 높게 설정하되, 사진 한 장 플로우가 장벽을 낮추므로 달성 가능한 수준.

### 리텐션 지표 목표

| 지표 | 초기 목표 (런치 후 3개월) | 성숙기 목표 (6개월+) |
|------|------------------------|-------------------|
| **D7 리텐션** | 25% | 35% |
| **D30 리텐션** | 12% | 20% |
| **월간 밑줄 생성** | 유저당 평균 4개/월 | 유저당 평균 8개/월 |
| **월간 공유 카드 생성** | 밑줄 저장 대비 20% | 밑줄 저장 대비 35% |

**근거**: 독서 앱의 특성상 방학·연초·독서의 달 시즌에 급증하고 평시에 떨어지는 패턴이 있다. D30 12%는 소셜 독서 앱 레퍼런스(Goodreads 초기) 대비 현실적인 목표이며, 리텐션 장치(오늘의 내 밑줄 알림, 완독 배지)가 붙으면 20%까지 상향 가능하다.

## 2. 채널별 콘텐츠 전략 (밑줄 특화)

### 인스타그램 북스타그램

**최적 공유 카드 포맷**

- 비율: 정사각형 1:1 (피드) + 세로 9:16 (스토리) 두 가지 생성 지원
- 추천 테마: `paper` (화이트 피드 독자 70%), `dark` (임팩트형), `photo` (진정성)
- 텍스트 길이: 본문 문장 20~50자 (맥락 독립적으로 서있는 문장)
- 필수 요소: 책 제목 + 저자명 + watermark(밑줄 앱명)
- 선택 요소: 책 표지 썸네일, 페이지 번호

**캡션 패턴 (3가지)**

1. **공감형**: `[문장 그대로 인용]` + 줄바꿈 + `ㅡ [책 제목], [저자]` + 빈 줄 + `#책밑줄 #북스타그램 #글귀스타그램 #[책제목해시태그]`
   - 예: `"그건 아무렇지도 않았다."` + `ㅡ 채식주의자, 한강` + `#책밑줄 #한강 #채식주의자 #북스타그램`

2. **정체성형**: `이 문장에서 책을 덮었다.` + 줄바꿈 + `[문장 인용]` + `ㅡ [책 제목]`
   - 예: `이 문장에서 책을 덮었다.` + `"..."`

3. **연결형**: `나만 이 문장에 밑줄 그은 게 아니었다.` + 줄바꿈 + `[문장 인용]`

**알고리즘 최적화**

- 저장 유도 카피: "나중에 다시 보고 싶은 문장이라면 저장해두세요" (저장율 3배 가중치)
- 캐러셀 형식: 한 책에서 3~5개 문장 묶어 올리면 저장률 22% 상승 (인사이트 리서치 근거)
- 릴스는 단일 이미지 대비 평균 참여율 6~7% vs 0.35% — 릴스 전환 필요 (밑줄 카드 슬라이드 + BGM)
- 발행 주기: 주 3~4회 (알고리즘 패널티 없는 최소 빈도)

### 틱톡 북톡

**밑줄 앱 소개 숏폼 훅 3가지 (구체적 스크립트)**

**훅 1: "진짜 독자 선별" 프레임**
```
[0~3초] 화면 텍스트: "세상엔 2종류의 독자가 있습니다"
[3~5초] 컷 전환: 아무 표시 없는 깨끗한 책 페이지
[5~7초] 컷 전환: 형광펜 밑줄 가득한 책 페이지 클로즈업
[7~12초] VO: "밑줄 긋는 독자는 책 속 문장을 어디에 저장하세요?"
[12~18초] 앱 화면 데모: 사진 → OCR → 카드 생성 10초 완성
[18~22초] CTA: "밑줄 앱에서 지금 바로 저장해보세요"
트렌딩 오디오 사용 (잔잔한 BGM 또는 책 페이지 넘기는 소리 ASMR)
```

**훅 2: "이 문장 때문에 울었다" 감성 쇼트**
```
[0~2초] 텍스트만: "이 문장 때문에 책을 내려놨다"
[2~6초] 밑줄 그어진 책 페이지 슬로모션 클로즈업
[6~10초] 폰에서 사진 찍는 손 + 앱에서 문장 추출되는 화면
[10~14초] 완성된 공유 카드 화면
[14~18초] CTA 텍스트 오버레이: "사진 한 장으로 충분합니다"
```

**훅 3: "같은 문장에서 멈춘 독자 N명" 발견 쇼트**
```
[0~3초] VO: "한강의 이 문장에 밑줄 그은 사람이 237명이에요"
[3~8초] 밑줄 상세 페이지 + "여기서 같이 멈췄어요" 섹션 화면
[8~12초] 독자 프로필 미리보기 슬라이드
[12~18초] VO: "당신의 밑줄 문장은 누군가의 오늘을 설명할 수 있어요"
[18~22초] CTA
```

**공통 틱톡 전략**

- 최적 문장 길이 오버레이: 8~11단어 (BookTok 데이터 기준)
- 업로드 시간: 화~목 19:00~21:00 (독서 인구 활동 시간)
- 해시태그: `#북톡` `#책추천` `#독서기록` `#밑줄` (4개 이하, 과적재 금지)
- 반드시 앞 3초에 훅 — "이 문장 때문에" / "책 읽다가" / "독자라면 공감" 中 1개 오프닝

### 네이버 블로그

**독서 에세이 + 자연스러운 앱 노출 전략**

포맷: 독서 에세이 형식 (1,500~2,500자), 앱 언급은 글 70% 지점 이후 1회

구조:
```
1. 책 읽은 경험 공감 서두 (300자)
2. 밑줄 그은 이유 + 문장 인용 (700자)
3. "이 문장을 어떻게 저장했는가" 자연스러운 앱 소개 (200자)
4. 앱 스크린샷 2~3장 (캡션에 기능 설명)
5. 같은 책의 다른 독자 발견 이야기 (300자)
6. 마무리 + 링크 (200자)
```

SEO 최적화 키워드:
- 주요: `책 밑줄 앱`, `독서 기록 앱`, `책 문장 저장`, `북스타그램 앱`
- 롱테일: `책 사진으로 문장 저장`, `형광펜 밑줄 디지털화`, `종이책 독서 앱`

발행 주기: 주 1회 (독서 관련 키워드 지속 누적)

금지 표현: "OCR 기술", "플랫폼", "독서량 관리" — 생산성 앱 연상 절대 금지

### 카카오 오픈채팅 독서 모임

**커뮤니티 진입 전략**

단계별 접근 (절대 첫 진입부터 홍보 금지):

| 단계 | 행동 | 기간 |
|------|------|------|
| 1단계 (관계 구축) | 독서 모임 오픈채팅 5~10개 참여, 2주간 가치있는 독서 이야기만 공유 | 2주 |
| 2단계 (자연 노출) | 대화 중 "사진 찍어서 저장해뒀어요" → 앱 화면 스크린샷 공유 | 3~4주차 |
| 3단계 (초대) | "같은 책 읽은 분들 밑줄 모아놓는 앱인데 같이 써볼 분?" | 5주차~ |

타겟 채팅방:
- "올해 책 100권 읽기" 챌린지 방 (연초 활성)
- 특정 책 완독 후기 방 (한강, 정세랑 등)
- 밀드라이너/형광펜 덕후 방 (물리적 밑줄 도구 사용자)
- 북클럽 정기 모임 방

핵심 메시지: "우리 모임 멤버들 밑줄 한 곳에 모아보면 재미있을 것 같아서요"

## 3. 유저 세그먼트별 활성화 전략

### 문장 수집가

**특성**: 좋은 문장을 메모앱·노션·사진 폴더에 혼자 저장해온 사람. 기존 방식에 마찰을 느끼지만 대안을 모름.

**활성화 전략**: OCR 품질로 첫 경험에서 감동 유발. "사진 찍으면 자동으로 글자 인식" 데모를 첫 화면에 배치.

**첫 메시지/캡션 예시**:
- 온보딩 첫 화면: `"지금까지 저장한 문장들, 사진 폴더 어딘가에 묻혀 있지 않으세요?"`
- 첫 사용 유도 푸시: `"책 읽다가 사진 찍어둔 게 있으면, 지금 불러와 보세요. 문장이 자동으로 추출됩니다."`
- 공유 유도: `"당신만 보기엔 아까운 문장이에요."`

**전환 지표**: 갤러리 기존 사진 업로드율 (신규 촬영 vs 기존 사진 업로드 비율)

### 북스타그래머

**특성**: 인스타그램에 독서 콘텐츠를 올리는 사람. 카드 퀄리티와 심미성에 민감. 팔로워 반응이 동기.

**활성화 전략**: 공유 카드 퀄리티 먼저 보여주기. `paper`, `dark`, `scene` 테마 선택지로 "내가 원하는 무드"를 만들 수 있음을 강조.

**첫 메시지/캡션 예시**:
- 첫 카드 생성 완료 화면: `"인스타에 올릴 준비 됐어요. 저장 또는 공유해보세요."`
- 테마 선택 화면: `"어떤 분위기로 공유하시겠어요?"`
- 카드 저장 후: `"#책밑줄 해시태그와 함께 올려보세요. 같은 책 읽은 독자들이 발견할 거예요."`

**전환 지표**: 첫 카드 생성율, 카드 외부 공유율(SNS 공유 버튼 탭 수)

### 필사러

**특성**: 손으로 옮겨 쓰는 독서 방식 선호. 디지털 전환에 다소 저항감. 블로그·노트에 기록하는 아날로그 감성 강함.

**활성화 전략**: "손글씨 필사를 대체하는 게 아니라 보완하는 도구"로 포지셔닝. 디지털화 후 손글씨 필사 이미지도 같이 올릴 수 있는 워크플로우를 보여줌.

**첫 메시지/캡션 예시**:
- 온보딩: `"필사한 문장을 사진으로 찍어도 됩니다. 손글씨도 인식해요."`
- 저장 완료: `"디지털 필사본이 만들어졌어요. 검색도 되고, 나중에 다시 볼 수도 있어요."`
- 공유 유도: `"이 문장, 당신만 알기엔 아까워요."`

**전환 지표**: 손글씨 이미지 OCR 성공율, 필사 관련 유저 세그먼트 D7 리텐션

### 소셜 독자

**특성**: 다른 독자의 밑줄이 궁금한 사람. 팔로우, 피드 탐색, 좋아요 행동이 활발. 책을 읽기보다 먼저 "다른 사람들의 반응"을 보는 유형.

**활성화 전략**: 공개 피드(/feed)와 책별 히트맵(/book/[id])을 비로그인 상태에서도 최대한 노출. "가입 없이 구경"을 허용하되, 좋아요/저장 시 가입 유도.

**첫 메시지/캡션 예시**:
- 비로그인 피드 하단 배너: `"이 문장에 밑줄 그은 독자가 몇 명인지 볼 수 있어요."`
- 좋아요 시도 시 로그인 모달: `"같이 멈춘 독자들과 연결되려면 가입이 필요해요."`
- 가입 완료 후 첫 화면: `"지금 읽고 있는 책이 있다면, 같은 책 독자들의 밑줄을 먼저 볼 수 있어요."`

**전환 지표**: 비로그인 피드 체류 시간, 좋아요 시도 → 가입 전환율

## 4. 연간 시즌 캠페인 플레이북

### 1월 연초 독서 결심 캠페인

**캠페인명**: "올해의 첫 밑줄"

**기간**: 1월 1일 ~ 1월 31일

**핵심 훅**: "올해도 좋은 책 읽겠다고 결심했다면, 처음 밑줄 그은 문장을 여기에 남겨보세요."

**채널별 실행**:
- 인스타그램: `#올해의첫밑줄` 해시태그 캠페인. 첫 저장 카드 공유 유도. 새해 첫 날 카운트다운 릴스.
- 틱톡: "2026년 첫 번째로 밑줄 그은 문장 공개" 챌린지 포맷. UGC 리포스트.
- 네이버 블로그: "연초 독서 계획 + 작년에 밑줄 그은 문장 결산" 포맷의 에세이.
- 카카오 채팅: "새해 첫 책 추천" 오픈채팅 방에서 밑줄 공유.

**KPI**:
- 신규 가입자 수: 런치 월 대비 200% 달성
- `#올해의첫밑줄` 해시태그 게시물: 500건 이상
- 1월 밑줄 생성 수: 전월 대비 300%
- 공유 카드 생성율: 밑줄 저장의 30% 이상

**주의**: 연초 결심 유저는 이탈 속도도 빠름. 2월까지의 리텐션 훅 준비 필수 (2월 1일 "당신의 1월 밑줄 결산" 이메일 발송).

### 6월 서울국제도서전 캠페인

## 5. 기획자에게 전달할 그로스 기반 기능 제안

### 리텐션 높이는 기능 Top 3

**1위: "오늘의 내 밑줄" 복습 알림 (D7/D30 리텐션에 직결)**

- **근거**: 밑줄은 "잊지 않으려고" 긋는다(기억 효과 d=0.36, 이해 효과 d=0.20의 2배). Readwise의 스페이스드 리피티션이 리텐션 핵심 기능인데, 밑줄 앱은 이를 단순화할 수 있다.
- **구현 방향**: 매일 오전 9시 "1년 전 오늘 / 1달 전 오늘 / 7일 전 오늘 당신의 밑줄" 푸시 알림 1개. 탭 시 해당 밑줄 상세 페이지 오픈.
- **예상 효과**: Readwise는 이 기능 하나로 구독 이탈율 60% 감소. 밑줄 앱은 무료이므로 리텐션 효과가 더 클 것으로 예상.
- **우선순위**: P0

**2위: 완독 배지 & "다 읽었어요" 선언 (재방문 + 공유 타이밍 포착)**

- **근거**: Hawking Index 분석에 따르면 완독률 높은 책(에세이·소설)에서 밑줄 공유율도 높다. 완독 직후가 공유 동기 최고점. 현재 앱은 이 순간을 포착하는 기능이 없다.
- **구현 방향**: `/book/[id]` 하단 "이 책 다 읽었어요" 버튼. 완독 선언 후 → "이 책에서 내 밑줄 전체 타임라인" + "공유 카드 만들기" 화면으로 자연스러운 유도.
- **예상 효과**: 완독 직후 공유 카드 생성율 +40%, 프로필 재방문율 +25%.
- **우선순위**: P1

**3위: 감정 태그 (피드 체류 시간 + 공유율 동시 향상)**

- **근거**: 바이럴 공식(Berger & Milkman)에 따르면 경이·그리움 같은 고각성 감정이 저각성 슬픔보다 바이럴 강도 3배. 현재 태그 시스템은 장르·주제 중심이지만 감정 레이어가 없다.
- **구현 방향**: `/new` 저장 단계에서 감정 칩 선택 (경이 / 그리움 / 공감 / 위로 / 충격 / 설렘 — 최대 1개, 선택 선택사항). 피드 필터에 감정 탭 추가.
- **예상 효과**: "경이" 태그 밑줄 저장률 DM 공유 +30%, 피드 체류 시간 +20%.
- **우선순위**: P1

### 바이럴 높이는 기능 Top 3

**1위: 캐러셀 공유 (저장률 22% 상승, 도달 확장)**

- **근거**: 인스타그램 캐러셀이 단일 이미지보다 저장률 22% 높음(리서치 데이터). 알고리즘이 저장을 좋아요보다 3배 가중. 완독 후 여러 문장 묶어 공유하는 행동 패턴에 딱 맞음.
- **구현 방향**: `/book/[id]` 또는 마이페이지에서 밑줄 2~6개 선택 → PNG 각각 생성 → ZIP 다운로드 또는 Web Share API.
- **예상 효과**: 외부 SNS 공유 유발 밑줄 수 +40%, `/book/[id]` 역유입 경로 다변화.
- **우선순위**: P1

**2위: 공유 카드 watermark + 랜딩 최적화 (비용 0, 효과 즉각)**

- **근거**: 현재 공유 카드에 앱 이름은 있지만 진입 URL이 명확하지 않으면 루프 단절. SNS 공유 카드가 바이럴 루프의 핵심인데 신규 유저 랜딩 경험이 최적화되어 있어야 함.
- **구현 방향**: 카드 하단에 `밑줄 앱 • underline.app` 형태 watermark. 공유 링크는 해당 밑줄 상세 페이지(`/underline/[id]`) 직링크. OG 이미지와 메타 태그 최적화.
- **예상 효과**: SNS → 앱 유입 전환율 +15%.
- **우선순위**: P0

**3위: "같은 문장 독자" 연결 강화 (소셜 그래프 형성)**

- **근거**: "이 문장에서 나만 멈춘 게 아니었구나"가 핵심 감정 훅(리서치에서 공유 동기 1위인 공감). 현재 "여기서 같이 멈췄어요" 섹션이 있지만 그 독자의 다른 밑줄로 이어지는 연결이 없음.
- **구현 방향**: 밑줄 상세에서 독자 아바타 탭 → 미니 프로필 슬라이드업 (그 독자의 밑줄 3개 미리보기). 팔로우 CTA.
- **예상 효과**: 프로필 페이지 유입 +35%, 팔로우 전환율 +20%.
- **우선순위**: P1

### 현재 가장 큰 활성화 장벽과 해결 방향

**장벽 1: "밑줄 저장 후 공유까지 가는 사람이 적다"**

- 현상: 저장은 되는데 카드 생성까지 안 가는 이탈
- 원인: 저장 완료 화면에서 공유 CTA가 약하거나 카드 생성 단계가 숨겨져 있음
- 해결: 저장 완료 화면에서 "공유 카드 만들기" 버튼을 1번 탭 이내로 배치. 카드 미리보기 썸네일 즉시 노출.

**장벽 2: "가입 후 첫 밑줄까지 마찰이 있다"**

- 현상: 가입은 했는데 첫 밑줄 저장을 안 하는 유저
- 원인: "지금 책을 읽고 있어야 한다"는 전제 조건. 갤러리에 이미 찍어둔 사진이 있어도 진입점이 불명확.
- 해결: 온보딩 마지막 단계에서 "갤러리에서 사진 불러오기" CTA. "이미 찍어둔 책 페이지 사진이 있으신가요?" 문구로 기존 행동 패턴과 연결.

**장벽 3: "D7 이후 재방문 동기가 없다"**

- 현상: 첫 밑줄 저장 후 앱에 돌아올 이유를 못 찾음
- 원인: 피드는 있지만 "나를 위한 콘텐츠" (내 밑줄 복습, 내가 읽은 책의 다른 독자 업데이트)가 없음
- 해결: "오늘의 내 밑줄" 복습 알림 (리텐션 기능 1위). 팔로우한 독자가 새 밑줄 저장 시 알림.

## 플래너에게 — 마케팅 인사이트 요약
