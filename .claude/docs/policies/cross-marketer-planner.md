## 기능-그로스 임팩트 매트릭스

백로그 및 로드맵 전체 기능 대상. 각 임팩트는 H(높음) / M(중간) / L(낮음)으로 평가.

| 기능 | 구현 상태 | 활성화 임팩트 | 리텐션 임팩트 | 바이럴 임팩트 | 최종 우선순위 |
|------|-----------|-------------|-------------|-------------|-------------|
| 밑줄 추가 (사진 OCR) | ✅ P0 완료 | H — 기록 행동의 진입점 자체 | H — 저장 경험이 재방문 동기 | H — 공유 카드의 원천 콘텐츠 | P0 기준선 |
| 공개 피드 (/feed) | ✅ P0 완료 | M — 가입 전 비로그인 노출 가능 | H — 새 콘텐츠 발견이 재방문 동기 | M — 피드 링크 공유 시 유입 | P0 기준선 |
| 좋아요 | ✅ P0 완료 | M — 가입 전 시도 → 가입 유도 전환점 | M — 좋아요 알림이 재방문 훅 | L — 바이럴 직접 기여 낮음 | P0 기준선 |
| 공유 이미지 카드 | ✅ P1 완료 | H — 카드 본 신규 유저의 앱 진입 경로 | M — 완성도 높은 카드가 재사용 동기 | H — 바이럴 루프의 핵심 매개체 | P0 수준으로 관리 필요 |
| 공감자 프로파일 ("여기서 같이 멈췄어요") | ✅ P1 완료 | L — 가입 전환 직접 기여 약함 | H — "나만 아니었다" 공명이 재방문 동기 | M — 공감자 발견 후 공유 유발 | P1 완료 |
| 책 단위 피드 (/books) | ✅ P1 완료 | M — 책 탐색 유저 진입 | M — 책 관심사 기반 재방문 | M — 특정 책 팬들 공유 | P1 완료 |
| 팔로우 DB 구현 | P1 잔여 (UI만) | M — 팔로우 후 팔로잉 피드 생성 → 활성화 완성 | H — 팔로잉 독자의 새 밑줄 알림이 가장 강력한 재방문 훅 | M — 팔로우 관계가 소셜 그래프 확장 | **P1 즉시 착수** |
| 책 히트맵 전면화 (/books 카드에 스니펫) | 미구현 | H — "가장 많이 밑줄 친 문장" 미끼로 비로그인 유저 유입 | M — 관심 책 업데이트 추적 동기 | H — "이 책 N명이 멈춘 문장" SNS 공유 시 입소문 | **P1 즉시 착수** |
| 공유 카드 watermark + OG 이미지 최적화 | 미구현 | H — 카드 클릭 → 앱 진입 경로 명확화, 전환율 직결 | L — 리텐션 직접 기여 없음 | H — 바이럴 루프 단절 방지, K-factor 직결 | **P0 수준 긴급** |
| "오늘의 내 밑줄" 복습 푸시 알림 | 미구현 | L — 기존 유저에게만 작동 | H — D7+ 재방문 이탈 방어 최강 장치 (Readwise 사례) | L — 바이럴 직접 기여 없음 | P1 고우선 |
| 완독 배지 & "다 읽었어요" 선언 | 미구현 | M — 완독 순간 공유 동기 포착 → 새 유저 활성화 | M — 완독 후 책 관련 밑줄 재탐색 동기 | H — 완독 카드는 SNS 공유율 높음 (Goodreads 사례) | P1 |
| 감정 태그 (경이/그리움/공감 등) | 미구현 | L — 저장 단계 추가 마찰 발생 위험 | M — 감정 필터 탐색이 피드 체류 시간 향상 | H — 감정 태그 카드는 공유율 +30% (Berger & Milkman) | P1 (단, 저장 마찰 최소화 설계 필수) |
| 캐러셀 공유 (여러 밑줄 묶음 PNG) | 미구현 | L — 기존 유저 활용 기능 | M — 책 완독 후 결산 콘텐츠 생성 동기 | H — 인스타 저장률 22% 상승 근거, 알고리즘 가중치 높음 | P1 |
| 온보딩 개선 (갤러리 사진 불러오기 CTA) | 미구현 | H — 가입 직후 첫 밑줄 저장 전환율 직결 | L — 리텐션 직접 기여 없음 | L — 바이럴 직접 기여 없음 | P1 (활성화 긴급) |
| 팔로우 알림 (새 밑줄 알림) | 미구현 (팔로우 DB 선행 필요) | L — 기존 관계 기반 | H — 팔로우 관계가 생기면 알림이 DAU 직결 | L | P1 (팔로우 DB 완료 후) |
| 추천 엔진 | P2 백로그 | L — 초기 데이터 부족으로 콜드 스타트 문제 | M — 데이터 충분 시 개인화로 재방문 | M — 추천 문장 공유 | P2 (MAU 10,000+ 이후) |
| 북클럽 / 댓글 / DM | P2 백로그 | L | M | L — 밀접 관계 형성, 직접 바이럴 낮음 | P2 |
| 연간 결산 Wrapped 카드 | P2 백로그 | L — 12월 한정 | H — 연간 회고로 장기 유저 재활성화 | H — Spotify Wrapped 효과, 12월 집중 공유 폭발 | P2 (단, 12월 전 준비 착수 필요) |

## "기록 vs 공유" 기능 균형 분석

### 현재 비중 평가

구현된 기능(P0 + P1 완료 11개)을 Planner 정책의 유틸리티/소셜 기준으로 분류하면:

**기록(유틸리티) 기능 — 5개 (45%)**
- 밑줄 추가 (사진 OCR)
- 책 자동 인식
- 페이지 번호 감지
- 마이페이지 (내 밑줄 모음)
- 크롭/회전 편집

**소셜 기능 — 6개 (55%)**
- 공개 피드 (/feed)
- 좋아요
- 공감자 프로파일
- 책 단위 피드 (/books)
- 공유 이미지 카드
- 책별 밑줄 모음 + 히트맵

**평가**: 수치상 소셜 비중이 약간 높지만, 소셜 기능의 절반(피드, 히트맵)이 기록 결과를 보여주는 방식이어서 실질적으로는 균형 잡힌 상태.

### 이상적인 기록:공유 기능 비율 (근거 포함)

**권장 비율: 6:4 (기록 우선)**

**근거**:
1. Planner 정책은 "기록 마찰 제거 > 소셜 기능 추가"를 명시. 기록이 없으면 공유할 콘텐츠가 없다.
2. 밑줄 서비스의 핵심 차별점은 종이책 OCR — 이 UX가 경쟁사 대비 우위이므로 기록 완성도에 투자 ROI가 높다.
3. 소셜 기능은 소셜 그래프(팔로우 관계)가 형성되어야 의미를 가진다. 현재 팔로우 DB도 미구현인 상태에서 소셜 기능을 추가해봐야 빈 관계 위에서 작동하지 않는다.
4. Marketer 정책의 활성화 장벽 분석 — "가입 후 첫 밑줄까지 마찰"이 가장 큰 이탈 원인. 기록 UX 개선이 K-factor 개선보다 먼저다.

**단, 예외**: 공유 카드 watermark + OG 최적화는 개발 비용이 극히 낮고 바이럴 루프 단절 방지 효과가 즉각적이므로 규칙 외 P0 수준으로 처리.

### 다음 이터레이션 방향 제안

현재 소셜 기능 비중이 이미 5:5에 근접하므로, 다음 이터레이션은 기록 UX 강화 1~2개 + 소셜 연결 1개 조합을 권장.

| 이터레이션 | 기록 강화 | 소셜 강화 | 균형 |
|-----------|-----------|-----------|------|
| 이번 스프린트 | 온보딩 개선 (갤러리 CTA) | 팔로우 DB | 5.5:4.5 |
| 다음 스프린트 | "오늘의 내 밑줄" 알림 | 책 히트맵 전면화 | 6:4 |
| 그 다음 | 감정 태그 (선택형, 마찰 최소) | 캐러셀 공유 | 6:4 유지 |

## 유저 세그먼트별 기능 로드맵

### 문장 수집가

**특성**: 메모앱·노션·사진 폴더에 문장을 저장해온 유저. 디지털 기록 습관 있음. 공유보다 수집에 방점.

**핵심 기능 (Marketer 분석 기반)**:

| 기능 | 이유 | 우선순위 |
|------|------|---------|
| OCR 정확도 유지·개선 | 첫 경험에서 OCR 감동이 없으면 전환 안 됨 | P0 상시 |
| 갤러리 기존 사진 업로드 (온보딩 CTA) | "이미 찍어둔 사진"과 연결 = 기존 습관 그대로 진입 | P1 긴급 |
| "오늘의 내 밑줄" 복습 알림 | 수집 → 재발견 루프 완성. Readwise 유저와 동일 니즈 | P1 |
| 밑줄 텍스트 검색 | 누적 문장이 많아질수록 검색 필요성 증가 | P2 (누적 50개+ 이후 의미) |
| 연간 결산 Wrapped | "올해 모은 문장들" 결산 — 수집가에게 최고 보상 | P2 |

**Aha 모멘트**: 처음 사진을 올렸을 때 OCR이 문장을 정확히 추출하는 순간. 이 경험 이전에 이탈하면 안 됨.

### 북스타그래머

**특성**: 인스타그램에 독서 콘텐츠를 올리는 유저. 카드 비주얼에 민감. 팔로워 반응이 동기.

**핵심 기능 (Marketer 분석 기반)**:

| 기능 | 이유 | 우선순위 |
|------|------|---------|
| 공유 카드 테마 시스템 (현재 6종) | 심미성이 공유 동기 = 카드 퀄리티가 이 세그먼트의 핵심 가치 | P0 완료, 지속 개선 |
| 공유 카드 watermark + OG 최적화 | 카드 공유 → 앱 유입 루프. 이 세그먼트가 생성하는 카드가 주요 획득 채널 | P0 수준 긴급 |
| 캐러셀 공유 | 인스타 북스타그램 포맷에 딱 맞음. 저장률 22% 상승 = 이 세그먼트의 알고리즘 최적화 | P1 |
| 완독 배지 + 공유 카드 | 완독 선언 후 한 권 전체 밑줄 카드 — 북스타그램 콘텐츠 포맷으로 최적 | P1 |
| 감정 태그 | 무드별 카드 필터링 → "오늘 슬픈 문장 카드" 등 큐레이션 콘텐츠 가능 | P1 |

**Aha 모멘트**: 첫 카드를 생성하고 "인스타에 올릴 수 있겠다"고 느끼는 순간. paper/dark 테마로 즉시 도달 가능.

### 공통 우선 기능

두 세그먼트(+필사러, 소셜 독자) 모두에게 영향을 미치는 기능:

| 기능 | 이유 | 우선순위 |
|------|------|---------|
| 팔로우 DB 구현 | 모든 세그먼트의 "발견 → 연결" 루프 완성. 소셜 그래프 없이는 재방문 동기 빈약 | P1 즉시 |
| 책 히트맵 전면화 | 수집가에게 "다른 독자의 밑줄 발견" 동기, 북스타그래머에게 "유행 책 문장 선점" 동기 동시 충족 | P1 즉시 |
| 온보딩 개선 (첫 밑줄 저장 CTA 강화) | 활성화 전환율 — 모든 세그먼트의 첫 번째 장벽 | P1 긴급 |
| "오늘의 내 밑줄" 복습 알림 | D7+ 재방문 이탈 방어 — 세그먼트 무관 리텐션 최강 장치 | P1 |

## 첫 번째 Aha 모멘트 개선 제안

### 현재 온보딩에서 Aha 모멘트가 언제 오는가 (분석)

현재 플로우:
```
1. 랜딩 / 로그인 화면
2. 회원가입 (이메일 또는 Google)
3. /new 페이지로 이동 (또는 피드로 이동)
4. 사진 업로드 → 크롭/회전
5. OCR 처리 중 (가장 긴 대기)
6. 책 정보 확인 + 밑줄 확인
7. 저장 완료
```

**현재 Aha 모멘트**: 6단계 — OCR이 사진에서 책 제목과 밑줄 문장을 정확히 추출했을 때. 하지만 이 순간에 도달하려면 "지금 책을 읽고 있어야 한다"는 전제 조건이 있다.

**문제점**:
- 가입 직후 /new로 유도되지 않거나, /new로 가더라도 올릴 사진이 없으면 이탈
- OCR 대기 중 불안감 (처리 상태 피드백 부족 시)
- 저장 완료 후 공유 카드로 이어지는 동선이 약해 Aha 모멘트가 저장에서 끝남 (공유 동기까지 연결 안 됨)
- Marketer 정책의 Time-to-First-Underline 목표(24시간 이내 70%)를 달성하기 위한 가입 직후 행동 유도 장치 없음

### 더 빠른 Aha 모멘트를 위한 구체적 기획 제안

**제안 1: 온보딩 마지막 단계에 "갤러리 불러오기" 즉시 CTA**

```
가입 완료 화면 → "지금 책 사진이 있으신가요?" 모달
   → [갤러리에서 불러오기] 버튼 (즉시 /new 플로우 진입)
   → [나중에 할게요] (피드로 이동)
```

- 근거: 밑줄 긋는 독자는 이미 찍어둔 책 사진이 갤러리에 있는 경우가 많음 (Marketer 정책의 "갤러리 기존 사진 업로드율" 지표 설정 근거와 일치)
- 예상 효과: Time-to-First-Underline 중위값 12시간 → 4시간 이내로 단축

**제안 2: 저장 완료 화면에서 공유 카드 미리보기 즉시 노출**

```
저장 완료 화면
  → 카드 미리보기 썸네일 즉시 표시 (기본 테마로 자동 생성)
  → [공유 카드 만들기] 버튼 1탭 이내 배치
  → [피드에서 보기] 버튼 병렬 제공
```

- 근거: 현재 저장 완료 → 카드 생성 단계 간 이탈이 크다 (Marketer 정책의 "장벽 1" 분석)
- 예상 효과: 첫 카드 생성율 +25%, 첫 공유 전환율 15% → 25%

**제안 3: 비로그인 피드를 Aha 유입 경로로 강화**

```
SNS에서 공유 카드 클릭 → /underline/[id] 상세 페이지
  → "여기서 같이 멈췄어요" 독자 N명 노출
  → [이 독자 팔로우] = 가입 CTA
  → 또는 [피드 전체 보기] = 비로그인 탐색 허용
```

- 근거: 소셜 독자 세그먼트는 "다른 사람들의 반응"을 먼저 보는 유형. 좋아요 시도 → 가입 유도 전환율이 이 플로우에서 가장 높음 (Marketer 정책의 소셜 독자 분석)
- 예상 효과: SNS → 앱 유입 → 가입 전환율 +15%

**제안 4: 온보딩 독자 타입 선택 (Marketer 인사이트 기반)**

```
가입 완료 → "어떤 독자인가요?" (1단계)
  → [밑줄 긋는 독자] → 즉시 /new로 유도
  → [다른 독자들 궁금한 독자] → /feed 또는 /books로 유도
```

- 근거: marketing-research.md의 "세상에 2종류" 프레임을 온보딩에 반영. 두 유형 모두 Aha 모멘트 경로를 최적화
- 예상 효과: 유저 타입별 활성화 경로 분기 → 전체 Activation Rate +10%p

## 마케터가 기획자에게 요청하는 기능 (우선순위 Top 5)

### 1위: 공유 카드 watermark + 랜딩 OG 이미지 최적화

**요청 이유**: 현재 바이럴 루프의 가장 약한 고리. 공유 카드가 SNS에서 노출되어도 앱 진입 경로가 불명확하면 K-factor가 0에 가까워진다. 공유 카드는 이미 구현되어 있으므로 개선 비용이 극히 낮다.

**예상 그로스 임팩트**:
- 공유 카드 → 앱 유입 전환율: 현재 추정 2~3% → 목표 8~12%
- K-factor: 현재 ~0.1 → 0.3+ (3배 향상)
- 신규 획득 채널 중 "카드 유입" 비중: 5% → 30%

**구현 우선순위**: P0 수준 긴급. 개발 공수 0.5일 이내. 이번 스프린트에서 반드시 포함.

---

### 2위: 온보딩 개선 — 갤러리 불러오기 CTA + 독자 타입 선택

**요청 이유**: Time-to-First-Underline이 활성화 North Star 지표. 현재 가입 후 첫 밑줄 저장 경험을 하지 못하는 유저가 이탈하는 것이 가장 큰 획득 비용 낭비. 온보딩 1단계 개선만으로 Activation Rate를 즉각 개선할 수 있다.

**예상 그로스 임팩트**:
- Activation Rate: 현재 추정 30% → 목표 50% (Marketer 정책 기준)
- Time-to-First-Underline: 중위값 12시간 → 4시간 이내
- D7 리텐션: 활성화 유저의 D7 리텐션이 미활성화 유저 대비 3~5배 높으므로 간접적으로 D7도 향상

**구현 우선순위**: P1 긴급. 개발 공수 1일. 팔로우 DB와 동시 진행 권장.

---

### 3위: 팔로우 DB 구현 + 팔로잉 피드 알림

**요청 이유**: 현재 "여기서 같이 멈췄어요" 공감자 프로파일이 있지만 팔로우가 안 된다. 공명 감정을 느낀 직후 팔로우 CTA가 없으면 관계가 형성되지 않고, D7+ 재방문 동기가 생기지 않는다. 소셜 그래프가 없는 소셜 앱은 리텐션 구조적 한계에 봉착한다.

**예상 그로스 임팩트**:
- D7 리텐션: 팔로잉 관계가 1개 이상인 유저의 D7 리텐션 목표 35% → 45%
- 프로필 페이지 유입: 공감자 프로파일 → 팔로우 CTA 연결 시 +35%
- DAU/MAU: 팔로잉 피드 알림 추가 시 0.10 → 0.15 예상

**구현 우선순위**: P1 즉시. 개발 공수 1~2일 (Planner 정책의 "즉시 착수 권장" 항목). 팔로우 UI는 이미 있으므로 DB + RLS + API만 추가하면 됨.

---

### 4위: "오늘의 내 밑줄" 복습 푸시 알림

**요청 이유**: 밑줄 저장 동기의 1순위는 "잊지 않으려고"이다. 그런데 저장 후 복습 기능이 없으면 저장 행위 자체의 효용이 감소하고, 앱에 돌아올 이유가 없어진다. Readwise가 이 기능 하나로 구독 이탈율을 60% 줄인 것이 근거. 밑줄 서비스는 무료이므로 심리적 장벽이 낮아 효과가 더 클 것.

**예상 그로스 임팩트**:
- D30 리텐션: 12% → 20% (목표치 조기 달성)
- MAU/DAU 비율: 0.10 → 0.18
- 월간 밑줄 저장 유지율: 알림 수신 유저의 월간 저장 수가 2배 이상 예상 (복습 → 새 밑줄 저장 동기 연쇄)

**구현 우선순위**: P1 고우선. 개발 공수 2~3일 (Web Push 인프라 포함). 팔로우 DB 완료 후 이어서 진행.

---

### 5위: 책 히트맵 전면화 — /books 카드에 "N명이 멈춘 문장" 스니펫

**요청 이유**: "/book/[id] 히트맵을 전면에 내세울 것"은 marketing-research.md에서 즉시 가능 P0으로 제안한 항목. "이 책의 독자들이 가장 많이 밑줄 친 문장"은 책 구매 전 독자들이 가장 원하는 정보이며, 이를 /books 피드에서 즉시 볼 수 있게 하면 비로그인 유저의 체류 시간과 가입 전환율이 동시에 올라간다. 또한 특정 책 관련 SNS 포스팅에서 "/books에서 이 책 밑줄 보기" 링크를 달면 퍼포먼스 마케팅 없이도 오가닉 유입 채널이 생긴다.

**예상 그로스 임팩트**:
- /books 페이지 비로그인 체류 시간: +40%
- /books → 가입 전환율: +15%
- 특정 책 관련 SNS에서 유입되는 오가닉 트래픽: 측정 가능한 레퍼러 생성

**구현 우선순위**: P1 즉시. 개발 공수 0.5일 (쿼리 + UI 변경만). 가장 ROI 높은 기능.
